# 지식 영양 성분표 분석기

사용자가 제공한 글을 Solar LLM으로 분석해 읽기 목적별 맞춤 '영양 성분표'를 출력하는 스킬입니다.

## 사전 준비 (최초 1회)

1. `assets/.env.example`을 `assets/.env`로 복사
2. `UPSTAGE_API_KEY=<키>` 설정 — 키 발급: https://console.upstage.ai
3. 의존성 설치: `pip install -r assets/requirements.txt`

## 실행 절차

### Step 1 — 입력 텍스트 확보

세 가지 입력 방식을 지원합니다.

**방식 A: 원문 직접 입력**
사용자가 글 텍스트를 그대로 붙여넣은 경우 — 해당 텍스트를 그대로 사용.

**방식 B: URL 입력**
사용자가 링크를 제공한 경우 — WebFetch 툴로 내용을 가져옵니다.
URL에서 가져온 HTML은 본문 텍스트만 추출 (네비게이션, 광고, 사이드바 제거).

**방식 C: 파일 첨부**
파일 경로를 받은 경우 — `--file` 인자로 스크립트에 전달.

### Step 2 — 모드 결정

사용자가 모드를 명시하지 않으면 다음 기준으로 추론하거나 물어봅니다:

| 상황 | 추천 모드 |
|------|-----------|
| "논문", "공부", "깊이 이해하고 싶어" | `bulkup` |
| "바빠", "핵심만", "결론이 뭐야" | `cutting` |
| "뉴스", "트렌드", "가볍게", "출퇴근길에" | `maintainer` |

### Step 3 — 분석 실행

텍스트를 `scripts/_input.txt`에 저장한 후 스크립트를 실행합니다:

```bash
python scripts/analyze.py <mode> --file scripts/_input.txt
```

예시:
```bash
python scripts/analyze.py bulkup --file scripts/_input.txt
python scripts/analyze.py cutting --file scripts/_input.txt
python scripts/analyze.py maintainer --file scripts/_input.txt
```

스크립트는 JSON을 반환합니다. 이 JSON을 파싱해 아래 형식으로 포매팅하세요.

### Step 4 — 결과 포매팅 및 출력

JSON 결과를 다음 마크다운 템플릿으로 변환합니다:
분석된 결과를 반드시 아래의 마크다운 템플릿 구조에 맞춰 변환하여 출력합니다.

**(🚨엄격한 출력 주의사항🚨)**
1. 제시된 템플릿 구조를 100% 그대로 유지하세요.
2. 템플릿 바깥이나 표 아래에 모델의 개인적인 분석, 주석, 부연 설명을 절대 덧붙이지 마세요.
3. 어떠한 섹션(표 포함)도 두 번 중복해서 출력하지 마세요. 오직 단 한 번만 출력해야 합니다.
```
## 🏷️ 지식 영양 성분표 — [모드명]

| 영양소 | 비율 (문장 수) | 주원료 (핵심 키워드) |
```

## 모드별 분류 규칙 (참고용)

### 벌크업 (Bulkup) — 깊은 학습
- **단백질**: 팩트(Data) + 통찰(Insight) 모두
- **탄수화물**: 이해를 돕는 비유/스토리
- **지방**: 의미 없는 잉여 문장만

### 컷팅 (Cutting) — 빠른 필터링
- **단백질**: 결론 포함 팩트(Data)만 — 통찰도 지방 취급
- **탄수화물**: 없음 (모든 설명은 지방)
- **지방**: 통찰, 스토리, 잉여 전부

### 유지어터 (Maintainer) — 가벼운 읽기
- **단백질/탄수화물**: 트렌드 인사이트 + 재미있는 비유
- **지방**: 전문 용어, 복잡한 데이터, 어려운 수식

### 공통 규칙
- **트랜스지방**: 모드 무관, 낚시성 제목 / 근거 없는 주장 / 논리적 비약

## 에러 처리

| 에러 | 대응 |
|------|------|
| `UPSTAGE_API_KEY not set` | `assets/.env` 파일 생성 안내 |
| 텍스트가 너무 짧음 (< 3문단) | 분석 가능하지만 결과가 부정확할 수 있다고 안내 |
| URL fetch 실패 | 사용자에게 텍스트 직접 붙여넣기 요청 |
| API 429 오류 | 잠시 후 재시도 안내 |
